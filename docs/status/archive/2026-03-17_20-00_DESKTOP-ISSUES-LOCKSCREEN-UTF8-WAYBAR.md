**Date**: 2026-03-17
**Severity**: High (breaks daily desktop workflow)
**Status**: Root causes identified, fixes documented

---

## Issues Investigated

1. SDDM not showing session/WM selector
2. Lock screen appearing automatically after ~5 minutes idle
3. Not all UTF-8 characters displaying (missing glyphs)
4. Waybar disappeared from Hyprland session

## 1. SDDM Not Letting You Choose Window Manager

**File**: `platforms/nixos/desktop/display-manager.nix:16-28`

**Root Cause**: The `sugar-dark` SDDM theme may not expose a visible session selector dropdown. Session files for Hyprland, niri, and Sway are all auto-generated by their respective NixOS modules and should be available, but the theme doesn't surface them.

**Current Config**:
```nix
services.displayManager.sddm = {
  enable = true;
  wayland.enable = true;
  theme = "sugar-dark";
  enableHidpi = true;
  autoNumlock = true;
};
```

**Fix Options**:
- Set a default session explicitly: `settings.General.Session = "hyprland";`
- Switch to a theme with better session selection (e.g., `sugar-candy`)
- Verify session files exist in `/usr/share/xsessions/` and `/usr/share/wayland-sessions/`

**Affected Sessions**:
- Hyprland (via `programs.hyprland.withUWSM = true`)
- niri (via `programs.niri.enable = true`)
- Sway (via `programs.sway.enable = true`)

## 2. Auto-Lock After 5 Minutes Idle

**File**: `platforms/nixos/programs/hypridle.nix`

**Root Cause**: Explicitly configured idle timeout chain:

| Timeout | Action | Config Line |
|---------|--------|-------------|
| 150s (2.5 min) | Dim screen to 30% brightness | `hypridle.nix:19` |
| 300s (5 min) | Lock screen (hyprlock) | `hypridle.nix:25` |
| 480s (8 min) | Turn off display (DPMS off) | `hypridle.nix:31` |
| 1200s (20 min) | Suspend system | `hypridle.nix:37` |

**Mitigation**: Waybar has an `idle_inhibitor` module (center bar, eye icon). Clicking it sends a D-Bus inhibit signal that should suppress all timeouts. However, `ignore_dbus_inhibit = false` in `hypridle.nix:12` means external inhibitors are respected — if the inhibitor isn't being picked up correctly, this won't work.

**Fix Options**:
- Increase timeouts (e.g., lock at 10-15 min instead of 5)
- Set `ignore_dbus_inhibit = true` to never allow inhibitors to be overridden
- Verify the idle_inhibitor module is working: `busctl --user call org.freedesktop.portal.Desktop /org/freedesktop/portal/desktop org.freedesktop.portal.Inhibit CreateSession`

## 3. Not All UTF-8 Characters Displaying

**Files**:
- `platforms/nixos/system/configuration.nix:98-102` (fontconfig defaults)
- `platforms/common/packages/fonts.nix` (installed font packages)

**Root Cause**: Font stack has limited Unicode coverage. Currently installed:

| Font | Coverage |
|------|----------|
| JetBrains Mono | Latin, Latin Extended, some symbols |
| Fira Code | Latin, Latin Extended, programming ligatures |
| Iosevka | Latin, Latin Extended, programming ligatures |
| DejaVu Sans/Serif | Latin, Cyrillic, Greek, basic symbols |
| Bibata cursors | Mouse cursors only |

**Missing coverage**: Emoji, CJK (Chinese/Japanese/Korean), Arabic, Devanagari, other non-Latin scripts, extended Unicode symbols.

**Fix**: Add Noto Fonts as fallbacks in `common/packages/fonts.nix`:
```nix
noto-fonts
noto-fonts-color-emoji
noto-fonts-cjk-sans
noto-fonts-extra
```

And configure fontconfig fallback chain in `configuration.nix`:
```nix
fonts.fontconfig.defaultFonts = {
  monospace = ["JetBrains Mono" "Noto Sans Mono"];
  sansSerif = ["DejaVu Sans" "Noto Sans"];
  serif = ["DejaVu Serif" "Noto Serif"];
  emoji = ["Noto Color Emoji"];
};
```

## 4. Waybar Disappeared

**File**: `platforms/nixos/desktop/waybar.nix` (Home Manager), `platforms/nixos/desktop/hyprland.nix:212`

**Root Cause**: XDG desktop portal conflict introduced by commit `54d59df` (niri addition). Two competing portal configurations now exist:

| Source | Portals Configured |
|--------|--------------------|
| `hyprland-config.nix:15-20` | `xdg-desktop-portal-hyprland`, `xdg-desktop-portal-gtk` |
| `niri-config.nix:4-8` | `xdg-desktop-portal-gnome` |

When both `xdg-desktop-portal-gnome` and `xdg-desktop-portal-gtk` are loaded alongside the Hyprland portal, it causes Wayland compositor detection issues. This can crash Waybar or prevent it from connecting to the compositor's IPC socket.

**Secondary Cause**: Waybar is launched via `exec-once` in `hyprland.nix:212`. If it crashes on startup (from portal conflict or a failing custom module), it will not restart automatically.

**Fix**:
1. Remove `xdg-desktop-portal-gnome` from `niri-config.nix` (conflicts with Hyprland portal)
2. Alternatively, make portal selection conditional based on active session
3. Restart Waybar manually to test: `waybar &`

**Git Blame**: Commit `54d59df` (`feat(desktop): add niri scrollable-tiling Wayland compositor`, 2026-03-17) added `niri-config.nix` with the conflicting portal config.
