## a) FULLY DONE

### 1. Root Cause Analysis
- **Identified:** The `update.sh` script's `get_source_hash()` function used `nix-prefetch-url --type sha256` which outputs **base32** format (e.g., `0mawk5sgb8d2v2xk867ivs97zg02wb36jqha5b6ark1b3bs6f365`)
- **Problem:** The `package.nix` file expects **SRI format** (`sha256-...` base64 with prefix)
- **Impact:** All automated updates would fail with hash mismatch errors, causing silent rollbacks to previous versions

### 3. Update Script Fix
**File:** `pkgs/crush-patched/update.sh`

**Changes:**
1. **Hash Format Conversion:** Added SRI conversion using `nix hash to-sri`
   ```bash
   hash_base32=$(nix-prefetch-url --type sha256 "$url" 2>&1 | tail -1)
   hash_sri=$(echo "$hash_base32" | xargs nix hash to-sri --type sha256 2>/dev/null)
   ```

2. **Format Validation:** Added regex validation to catch conversion failures early
   ```bash
   if [[ ! "$hash_sri" =~ ^sha256-[A-Za-z0-9+/=]+$ ]]; then
     log_error "Invalid SRI hash format: $hash_sri"
     return 1
   fi
   ```

3. **Sed Pattern Fix:** Changed pattern from `sha256:.*` to `sha256-.*` to match SRI format correctly
   ```bash
   # Before (broken):
   sed -i.bak "s|hash = \"sha256:.*\";|hash = \"sha256:${hash}\";|g"
   
   # After (fixed):
   sed -i.bak "s|hash = \"sha256-.*\";|hash = \"${hash}\";|g"
   ```
