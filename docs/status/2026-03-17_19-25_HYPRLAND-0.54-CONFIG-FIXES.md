# Hyprland 0.54 Configuration Update Report

**Date:** 2026-03-17
**Component:** NixOS Desktop Environment (Hyprland)
**Status:** ✅ Completed

## Overview

The user reported a list of active configuration errors appearing in the Hyprland desktop environment, captured in a screenshot. The errors were related to deprecated syntax and invalid dispatchers that emerged following an update to Hyprland 0.54.2.

## Issues Identified

1. **Deprecated `windowrulev2` Syntax**: Hyprland 0.54 removed the legacy `windowrulev2` syntax in favor of the new unified `windowrule` block utilizing `match:` syntax.
2. **Invalid Dispatchers (hy3 plugin)**: The `hy3` plugin (used for i3-like layout management) was previously disabled in `hyprland.nix` due to incompatibility with Hyprland 0.54.2. However, the associated keybindings (`hy3:changegroup` and `hy3:makegroup`) remained in the configuration, causing "Invalid dispatcher" errors.

## Actions Taken

1. **Configuration Migration (`windowrulev2` -> `windowrule`)**:
   - Rewrote over 50 window rules in `platforms/nixos/desktop/hyprland.nix` from the old format:
     `"float, class:^(kitty)$"`
   - To the new format:
     `"float on, match:class ^(kitty)$"`
   - Translated other actions dynamically (e.g., `noborder` to `border_size 0`, `nofocus` to `no_focus on`, `noblur` to `no_blur on`, `noshadow` to `no_shadow on`).

2. **Removed Obsolete Bindings**:
   - Stripped all `hy3:changegroup` and `hy3:makegroup` keybindings (Super+W, Super+Y, Super+E, Super+Shift+E) since the `hy3` plugin is disabled.

3. **Type Safety Framework Update**:
   - Updated the reference mapping inside `platforms/nixos/desktop/hyprland.nix` (line 468) to map `settings.windowrule` rather than the removed `settings.windowrulev2` for the type validation module.

4. **Validation**:
   - Ran `just test-fast` to verify that the Nix evaluation completed successfully without breaking the type safety assertions defined in `HyprlandTypes.nix`.
