## Key Design Decisions

- **Mutex discipline:** `sync.RWMutex` for state reads, separate `cmdMu` for command serialization
- **Context propagation:** All I/O methods accept `context.Context`; HID sleep respects cancellation
- **Atomic state saves:** Write to `.tmp` then `os.Rename` to prevent corruption
- **PTZ caching:** 2-second TTL cache avoids redundant v4l2-ctl calls
- **Frame sharing:** MJPEG stream frames cached for snapshot endpoint (zero extra ffmpeg)
- **Stream semaphore:** Limits concurrent MJPEG streams to 1
- **UEVENT hotplug:** Netlink listener detects device plug/unplug, auto-syncs state
- **Security headers:** CSP, X-Frame-Options, Referrer-Policy, X-Content-Type-Options
- **Graceful shutdown:** SIGHUP saves state, SIGTERM cleans up socket + HTTP server with timeout
- **Request ID middleware:** X-Request-ID header for tracing
- **MaxBytesReader:** All POST endpoints limit request body size
- **PTZ validation:** Server-side clamping of pan/tilt/zoom to hardware limits
- **Build tags:** `//go:build linux` on all platform-specific files

## What Could Still Be Improved

### High value

1. **Test coverage for handlers.go** — 55.2% overall; handlers have integration tests but limited unit test coverage for streaming, PTZ parsing
2. **Fuzz testing for `parseHIDResponse`** — parses raw bytes from hardware; could be hardened
3. **Fuzz testing for `extractJPEGFrame`** — byte-level JPEG parsing; safety-critical for streaming

### Medium value

4. **Context cancellation in `Run()` ticker loop** — ticker case doesn't select on ctx.Done()
5. **WebSocket support** — currently htmx SSE/polling; WebSocket would be more efficient for real-time
6. **Prometheus `/metrics` endpoint** — for production observability
7. **Error type hierarchy** — HID errors are sentinel values; typed errors would enable programmatic handling

### Low value / future

8. **Replace `exec.Command` v4l2 calls with direct ioctls** — eliminates gosec false positives and subprocess overhead
9. **Structured event logging** — replace slog calls with typed event types
10. **API versioning for socket protocol** — string-based protocol has no version negotiation
